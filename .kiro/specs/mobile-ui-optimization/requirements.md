# 移动端UI优化需求文档

## 📋 项目概述

**项目名称**: CuckooX移动端UI优化  
**目标页面**: https://dev.cuckoox.cn/cases (案件管理页面)  
**更新时间**: 2025年1月27日  
**文档版本**: v1.0

### 项目背景
基于对现有破产案件管理系统案件列表页面的深入分析，发现在移动端存在严重的可用性问题：
- 表格布局在小屏幕上导致严重横向滚动
- 触摸目标过小，操作困难
- 信息密度过高，阅读体验差
- 缺乏移动端特有的交互模式

### 项目目标
将桌面端优先的设计转换为移动端友好的响应式界面，显著提升移动端用户体验。

## 🎯 核心需求领域

### 1. 移动端列表展示优化

**需求描述**: 将传统的7列表格在移动端转换为卡片式布局，确保信息清晰可读。

**用户故事**:
- 作为移动端用户，我希望能够在不横向滚动的情况下查看案件列表
- 作为用户，我希望能够快速识别案件的关键信息（编号、状态、负责人）
- 作为用户，我希望能够展开卡片查看详细信息

**验收标准**:
- [ ] 移动端显示卡片列表而非表格
- [ ] 每个卡片包含案件编号、状态、程序类型、负责人、时间信息
- [ ] 支持卡片展开/收起功能显示详细信息
- [ ] 卡片高度自适应内容，最小高度80px
- [ ] 卡片间距8px，左右边距16px

**信息层次设计**:
- **高优先级** (始终显示): 案件编号、当前状态、程序类型
- **中优先级** (可展开显示): 负责人、创建时间、更新时间
- **低优先级** (详情页查看): 创建人、详细描述、扩展属性

### 2. 触摸友好的交互设计

**需求描述**: 确保所有可交互元素符合移动端触摸标准，提供清晰的视觉反馈。

**用户故事**:
- 作为移动端用户，我希望能够准确点击操作按钮而不会误触
- 作为用户，我希望点击后有明确的视觉反馈
- 作为用户，我希望有快速创建新案件的便捷方式

**验收标准**:
- [ ] 所有可点击元素最小尺寸44px×44px
- [ ] 操作按钮间距最小8px避免误触
- [ ] 添加浮动操作按钮(FAB)用于创建新案件
- [ ] 所有触摸操作提供0.2s的视觉反馈动画
- [ ] 支持长按显示上下文菜单

**FAB设计规范**:
- 位置: 右下角固定，距离边缘16px
- 尺寸: 56px×56px
- 颜色: 主题青绿色 (#1BA2A0)
- 图标: mdiPlus，24px
- 阴影: elevation 6

### 3. 移动端搜索和筛选优化

**需求描述**: 重新设计搜索和筛选界面，适配移动端操作习惯和屏幕尺寸。

**用户故事**:
- 作为移动端用户，我希望有全屏的搜索界面，方便输入和查看结果
- 作为用户，我希望筛选选项以抽屉形式展示，便于选择和操作
- 作为用户，我希望看到当前激活的筛选状态

**验收标准**:
- [ ] 搜索框在移动端占据全屏宽度
- [ ] 筛选面板采用底部抽屉设计
- [ ] 显示当前激活筛选的数量badge
- [ ] 支持搜索历史和自动建议
- [ ] 提供快速清除筛选的功能

**筛选面板规范**:
- 从底部滑出，高度占屏幕的60%
- 支持多种筛选类型：单选、多选、日期范围
- 提供"重置"和"应用"按钮
- 滑动手势关闭面板

### 4. 统计信息移动端适配

**需求描述**: 调整统计卡片布局和内容，适应移动端屏幕空间限制。

**用户故事**:
- 作为移动端用户，我希望统计信息在小屏幕上也能清晰展示
- 作为用户，我希望能够点击统计卡片快速筛选对应数据
- 作为用户，我希望看到关键统计指标而不被次要信息干扰

**验收标准**:
- [ ] 移动端使用2×2网格布局，平板以上使用4列布局
- [ ] 统计卡片高度减少至80px (紧凑模式)
- [ ] 移动端隐藏趋势图表，仅显示数值和简单图标
- [ ] 每个卡片支持点击跳转到对应筛选视图
- [ ] 加载状态使用骨架屏显示

**紧凑模式设计**:
- 标题字体12px，数值字体24px加粗
- 图标16px，位于左上角
- 内边距8px，外边距4px
- 支持深色主题

### 5. 移动端导航优化

**需求描述**: 优化导航栏布局和交互，提供移动端专用的导航体验。

**用户故事**:
- 作为移动端用户，我希望导航栏固定在顶部，不随滚动消失
- 作为用户，我希望导航能够智能显示/隐藏以节省屏幕空间
- 作为用户，我希望页面切换有流畅的转场动画

**验收标准**:
- [ ] 顶部导航栏固定定位，高度56px
- [ ] 向下滚动时导航栏智能隐藏，向上滚动时显示
- [ ] 页面标题支持长文本的滚动显示
- [ ] 添加页面转场动画（左右滑动效果）
- [ ] 支持PWA模式下的安全区域适配

**导航栏规范**:
- 背景色：深色主题 #1A1A1A，浅色主题 #FFFFFF
- 标题字体16px加粗，居中显示
- 左侧返回按钮44px×44px
- 右侧操作按钮最多2个

### 6. 响应式布局系统

**需求描述**: 建立完整的响应式布局系统，支持各种设备尺寸和屏幕方向。

**用户故事**:
- 作为用户，我希望在不同设备上都有一致且优化的体验
- 作为用户，我希望横屏模式下能更好地利用屏幕空间
- 作为用户，我希望界面能够智能适配各种屏幕尺寸

**验收标准**:
- [ ] 支持手机(xs)、平板(md)、桌面(lg+)三种主要断点
- [ ] 横屏模式下统计卡片使用单行布局
- [ ] 组件能够根据可用空间智能调整显示密度
- [ ] 支持动态字体大小缩放
- [ ] 在不同设备上保持设计一致性

**断点设计**:
- xs: 0-599px (手机竖屏)
- sm: 600-959px (手机横屏/小平板)
- md: 960-1279px (平板)
- lg: 1280-1919px (桌面)
- xl: 1920px+ (大屏桌面)

### 7. 性能和加载优化

**需求描述**: 确保移动端加载速度和操作响应速度达到优秀标准。

**用户故事**:
- 作为移动端用户，我希望页面能够快速加载完成
- 作为用户，我希望滚动大量数据时界面保持流畅
- 作为用户，我希望在网络较慢时也能看到加载进度

**验收标准**:
- [ ] 首屏加载时间<3秒 (3G网络)
- [ ] 实现虚拟滚动处理>100条数据的列表
- [ ] 使用骨架屏替代传统loading动画
- [ ] 图片和非关键资源懒加载
- [ ] 滚动帧率保持>50fps

**性能指标**:
- Lighthouse Performance分数 >90
- First Contentful Paint <1.5s
- Largest Contentful Paint <2.5s
- Cumulative Layout Shift <0.1

### 8. 可访问性和用户体验

**需求描述**: 确保移动端界面满足无障碍访问标准，提供包容性的用户体验。

**用户故事**:
- 作为视觉障碍用户，我希望能够使用屏幕阅读器正常浏览
- 作为有运动障碍的用户，我希望有足够大的触摸目标
- 作为在强光环境的用户，我希望有高对比度模式

**验收标准**:
- [ ] 通过WCAG 2.1 AA级别标准
- [ ] 支持屏幕阅读器导航和语音播报
- [ ] 提供高对比度主题选项
- [ ] 支持键盘导航 (外接键盘)
- [ ] 提供文字大小调节功能

**无障碍规范**:
- 所有交互元素有合适的aria-label
- 颜色对比度比例>4.5:1
- 支持Tab键导航顺序
- 重要信息不仅依赖颜色传达

### 9. 数据展示优化

**需求描述**: 优化数据在移动端的展示方式，提高信息的可读性和可理解性。

**用户故事**:
- 作为移动端用户，我希望通过图标快速理解数据含义
- 作为用户，我希望时间信息以相对时间显示更加友好
- 作为用户，我希望通过颜色编码快速识别不同状态

**验收标准**:
- [ ] 状态信息使用颜色编码的Chip组件
- [ ] 时间显示相对时间（如"2小时前"、"昨天"）
- [ ] 程序类型使用图标+文字的形式
- [ ] 重要程度用不同颜色的指示点标记
- [ ] 数值信息支持千分位分隔符

**视觉编码规范**:
- 立案: 蓝色 (#1976D2)
- 进行中: 橙色 (#F57C00)
- 已完成: 绿色 (#388E3C)
- 终结: 灰色 (#616161)

### 10. 手势和交互增强

**需求描述**: 实现移动端特有的手势操作，提供更加自然和高效的交互体验。

**用户故事**:
- 作为移动端用户，我希望能够左滑显示快捷操作菜单
- 作为用户，我希望能够下拉刷新获取最新数据
- 作为用户，我希望能够长按进入多选模式

**验收标准**:
- [ ] 支持左滑显示操作菜单（查看、编辑、删除）
- [ ] 支持下拉刷新数据
- [ ] 长按卡片进入多选模式
- [ ] 双击卡片快速进入详情页
- [ ] 支持收藏/取消收藏的右滑手势

**手势规范**:
- 左滑距离>40px触发操作菜单
- 下拉距离>60px触发刷新
- 长按时间>500ms进入多选
- 双击间隔<300ms识别为双击

## 🔬 测试策略

### 功能测试
- **设备兼容性**: iPhone SE, iPhone 12 Pro, iPad, Android各主流机型
- **浏览器兼容性**: Safari, Chrome, Firefox移动版
- **操作系统**: iOS 14+, Android 8+
- **网络环境**: 4G, 3G, WiFi

### 用户测试
- **目标用户**: 5名法律行业移动端用户
- **测试任务**: 查看案件列表、搜索特定案件、查看案件详情、创建新案件
